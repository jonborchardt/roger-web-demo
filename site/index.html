Roger

High-resolution art for classic Sierra SCI adventure games — generated in-engine from the original game resources, with the game logic left completely untouched.

📱 Playing on a phone — read this first

The demo runs on phones, but be warned it is not fully optimal: Betrayed Alliance is a keyboard-and-parser adventure built for a desktop. Hold your phone in landscape. A side action bar gives you the on-screen keyboard plus Menu (Esc), Enhanced (F10), and a ↑ ← ↓ → / OK pad — the game's menu opens with Menu and is navigated with those arrows. The game shrinks to stay visible above the keyboard.

The one tip that makes it comfortable: put your keyboard in floating mode. The phone keyboard normally docks at the bottom and covers the text as you type. Most Android keyboards (e.g. Gboard) can float: open the keyboard, tap the ⋮ / toolbar menu, choose Floating (or Modes → Floating), then drag it out of the way — like this:

🖥️ Run it on your own computer

For the best experience — full keyboard and mouse — run the enhanced version natively on Windows, macOS, or Linux. Roger is a fork of ScummVM, so you build it from source:

➜ github.com/jonborchardt/scummvm (branch jon-wasm).

  • Windows: clone the repo and run build_and_run.ps1 — it fetches the libraries via vcpkg and builds with Visual Studio (Desktop C++ workload) on the first run.
  • macOS / Linux: the standard ScummVM build — ./configure && make — then launch the scummvm binary.

Point it at a supported SCI0 EGA game (SQ3, QFG1 EGA) or Betrayed Alliance, which is free. Roger is opt-in and, switched off, behaves byte-for-byte like stock ScummVM.

Note: ready-to-run downloads aren't published yet — for now it's a source build. Pre-built binaries are on the to-do list.

What it is

Roger is a display-layer enhancement inside ScummVM's SCI engine. It runs the original commercial games from their original data and re-renders the backgrounds and sprites at high resolution, while all game logic still runs at the native 320×200 resolution. It is opt-in and, when switched off, the engine behaves byte-for-byte like stock ScummVM.

How it works

Each room's native picture, priority, and control buffers render exactly as SCI always does — so walkability and occlusion "just work." Roger then generates a high-resolution plate from the same SCI resources (an in-engine upscaling pipeline) and composites it, plus upscaled sprites and the game's dialogs, through ScummVM's OSystem overlay. Nothing is pre-shipped as art; it's all derived from the game's own resources at run time.

Press F10 in the demo to cycle Enhanced → Original → Side-by-Side and compare for yourself.

For the full story of the upscaling pipeline, read the blog post: OMYAC Upscaler — leaning all the way into the anchor-graph approach.
